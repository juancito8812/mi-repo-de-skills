# Instrucciones para Codebuff

## Al iniciar cada sesión

1. **Revisa este repositorio** — Lee el README.md y familiarízate con todas las skills disponibles.
2. **Carga las skills relevantes** — Usa el tool `Skill` para cargar skills cuando apliquen al contexto de la conversación.
3. **Prioriza estas skills según el tipo de tarea:**

   | Tipo de tarea | Skills a cargar primero |
   |---------------|------------------------|
   | Diseño/planificación | brainstorming → writing-plans |
   | Implementación | test-driven-development → subagent-driven-development |
   | UI/Frontend | frontend-design, interface-design |
   | APIs REST | restful-api-design |
   | APIs GraphQL | graphql-api-design |
   | Bases de datos | postgresql-table-design |
   | Manejo de errores | error-handling-patterns |
   | React/Next.js | vercel-react-best-practices |
   | Debugging | systematic-debugging |
   | Code review | requesting-code-review |
   | Git/Changelog | auto-sync, changelog-generator |
   | Minimizar código | ponytail |

4. **Sigue el flujo de Superpowers:** Invoca skills ANTES de actuar, no después. "Incluso 1% de probabilidad = invocar el skill."
